# PROMPT PENGEMBANGAN SISTEM MANAJEMEN PROYEK

Lakukan pengembangan pada aplikasi Manajemen Proyek Laravel yang sudah ada dengan spesifikasi berikut:

## 1. PERUBAHAN FORM TAMBAH PROYEK

Saat membuat proyek baru:

### Progress Awal

* Field Progress tidak perlu diinput oleh user.
* Progress otomatis dibuat **0%** saat proyek dibuat.

### Rencana Pekerjaan

Tambahkan field:

* Rencana Pekerjaan

  * Textarea
  * Wajib diisi

* Rencana Durasi Pekerjaan

  * Input Number
  * Satuan Hari
  * Wajib diisi

Contoh:

```
Pembangunan Gudang A 1 berapa hari material nya ada apa aja
Durasi : 30 Hari
```

### Perhitungan Tanggal Otomatis

Field:

* Tanggal Mulai

Saat user memilih tanggal mulai:

```
Tanggal Selesai =
Tanggal Mulai + Durasi Hari
```

Contoh:

```
Tanggal Mulai : 01-06-2026
Durasi : 30 Hari

Tanggal Selesai otomatis :
01-07-2026
```

Tanggal selesai dapat dihitung otomatis menggunakan Javascript tanpa reload halaman.

# 3. MASTER PEKERJAAN

Buat CRUD lengkap:

### Tabel Master Pekerjaan

Field:

| Field          | Tipe      |
| -------------- | --------- |
| id             | bigint    |
| kode_pekerjaan | string    |
| nama_pekerjaan | string    |
| kategori       | string    |
| keterangan     | text      |
| created_at     | timestamp |

Contoh:

| Kode  | Nama             |
| ----- | ---------------- |
| PK001 | Finishing        |
| PK002 | Penambahan Bahan |
| PK003 | Struktur         |
| PK004 | Pengecatan       |

---

# 4. DETAIL PEKERJAAN

Setiap Master Pekerjaan memiliki banyak Detail Pekerjaan.

### Relasi

```
Master Pekerjaan
   hasMany
Detail Pekerjaan
```

Field:

| Field                 | Tipe        |
| --------------------- | ----------- |
| id                    | bigint      |
| master_pekerjaan_id   | foreign key |
| nama_detail_pekerjaan | string      |
| satuan                | string      |
| keterangan            | text        |

Contoh:

Master = Finishing

Detail:

```
Pemasangan Keramik
Pengecatan Dinding
Pemasangan Plafon
Pembersihan Akhir
```

---

# 5. DETAIL MATERIAL PER DETAIL PEKERJAAN

Setiap Detail Pekerjaan dapat memiliki banyak material.

### Relasi

```
Detail Pekerjaan
    hasMany
Detail Material
```

Field:

| Field               | Tipe        |
| ------------------- | ----------- |
| id                  | bigint      |
| detail_pekerjaan_id | foreign key |
| nama_material       | string      |
| spesifikasi         | string      |
| qty                 | decimal     |
| satuan              | string      |
| keterangan          | text        |

Contoh:

Detail Pekerjaan:

```
Pengecatan Dinding
```

Material:

```
Cat Dulux 25 Kg
Kuas
Roll Cat
Thinner
Lakban Kertas
```
